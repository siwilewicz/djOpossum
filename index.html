Let's start with HTML Table elements!

Table Tags

TAG NAME DESCRIPTION
< table > Table The wrapper element for all HTML tables
< thead > Table head The set of rows defining the column headers in a table
< tbody > Table body The set of rows containing actual table data
< tr > Table row The table row container
< td > Table data Table row container
< tfoot > Table foot The set of rows defining the footer in a table

Table Attributes

ATTRIBUTE NAME DESCRIPTION
< colspan > Column Defines how many columns a < td > element should span
< rowspan > Row span Defines how many rows a < td > element should span
< caption > Caption Definsed a table caption. Must be inserted immediatley after the < table > tag

HTML Attribute Reference

TAG DESCRIPTION
< a > Defines a hyper link, which is used to link from one page to another. Don't forget to use href attribute, which indicates the link's destination.
< article > Self-contained tag. An article should make sense on its own and it should be possible to distribute it independently from the rest of the site. Great for forum/blog post or a news story.
< aside > Defines content aside from the page content and should be indirectly related to the surrounding content.
< audio > Used to embed sound content in a document, such as music or other audio streams. If contains one or more < suorce > tags with different audio the browser will choose the first source it supports.
< b > Make text bold without marking it as important
< blockquote > Specifies a section that is quoted from another source. Dont forget to specifie cite="" in the opening tag!
< body > Contains all the contents of an HTML document
< br > Line break element
< button > Defines a clickable button. Inside a < button > element you can put text (and tags like < i >, < b >, < strong >, < br >, < img >, etc.). That;s not possible wirh a button created with the < input > element!! Always specify the type attribute for a < button > element to tell browser what type of button it is.
< div > Defines a division or a section in an HTML document. Used as a container for HTML elements - which is then styled with CSS or manipulated with JavaScript.
< dl > Description list. The < dt > tag defines a term/name and < dt > tag
< em > Used to define emphasized text. The content inside is typically displayed in italic.
< figcaption > Defines a caption for a < figure > element
< figure > Specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.
< footer > Defines a footer for a document or section. Typically contains authorship information, copyright information, contact information, sitemap, back to top links, related documents
< form> Used to create an HTML form for user input. Can contain one or more of the following form elements: < input >, < textarea >, < button >, < select >, < option >, < optgroup >, < fieldset >, < label >, < output >
< h1 > to < h6 > Used to define HTML headings.< h1 > defines the most important headind and < h6 > the least important heading.
< head > Container for metadata (data about data) and is placed between the < html > tag and the < body > tag.
< header > Represents a container for introductory content or a set of navigational links. Typically contains h1-h2 tags, logo or icon, authorship information.
< img > Used to embed an image in an HTML page. Required attributes: src - Specifies the path to the image alt - Specifies an alternate text for the image, if the image for some reason cannot be displayed
< input > Specifies an input field where the user can enter data. Can be displayed in several ways, depending on the type attribute (button/email/number/text/password/etc.).
< li > Defines a list item. Used inside ordered lists < ol > (displayed with numbers or letters), unordered lists < ul > and menu lists < menu > (both usually displayed with bullet points).
< link > Most often used to link to external style sheets. Remember about rel="" and href="".
< nav > Defines a set of navigation links (ONLY for major block navigation links).
< p > Defines a paragraph. Browsers automatically add a single blank line before and after each < p > element
script Used to embed a JavaScript.
< source > Used to specify multiple media resources for media elements, such as video/audio/picture. It allows you to specify alternative video/audio/image files which the browser may choose from, based on browser support or viewport width.
< span > Inline container used to mark up a part of a text, or a part of a document. Easily styled by CSS or manipulated with JavaScript using the class or id attribute.
< strong > Used to define text with strong importance. The content inside is typically displayed in bold
< title > Defines the title of the document. The title must be text-only, and it is shown in the browser's title bar or in the page's tab.
< video > Used to embed video content in a document, such as a movie clip or other video streams.
The comment tag is used to insert comments in the source code. Comments are not displayed in the browsers. You can use comments to explain your code, which can help you when you edit the source code at a later date. This is especially useful if you have a lot of code.

CSS properties

PROPERTY DESCRIPTION
background-color Specifies the background color of an element
background-image Specifies one or more background images for an element
border A shorthand property for border-width, border-style and border-color
border-color Sets the color of the four borders
border-style Sets the style of the four borders
box-sizing Defines how the width and height of an element are calculated: should they include padding and borders, or not
color Sets the color of text
cursor Specifies the mouse cursor to be displayed when pointing over an element
float Specifies whether an element should float to the left, right, or not at all
@font-face A rule that allows websites to download and use fonts other than the "web-safe" fonts
font-family Specifies the font family for text
font-size Specifies the font size of text
font-style Specifies the font style for text
font-weight Specifies the weight of a font
height Sets the height of an element
@import Allows you to import a style sheet into another style sheet
letter-spacing Increases or decreases the space between characters in a text
margin Sets all the margin properties in one declaration (-top/-bottom/-left/-right)
@media Sets the style rules for different media types/devices/sizes
min-height Sets the minimum height of an element
min-width Sets the minimum width of an element
opacity Sets the opacity level for an element. From 0 to 1
overflow Controls what happens to the content taht spills, overflows outside its box (hidden/scroll/visible/etc.)
padding can be more specipy like -bottom/-left/-right/-top/etc.
position Specifies the type of positioning method used for an element (static, relative, absolute or fixed)
text-align Specifies the horizontal alignment of text (center/left/right/justify)
text-transform Controls the capitalization of text (uppercase/lowercase/capitalize)
visibility Specifies whether or not an element is visible (hidden/visible)
width Sets the width of an element
word-spacing Increases or decreases the space between words in a text
z-index Controls how far back/forward an element should appear on the web page when elements overlap. z-index doesn't work on static elements.
